# Штаб ЛС

Локальное однопользовательское приложение для учёта полётных смен, рабочего времени и отдыха. Код размещается на GitHub Pages, рабочие данные сохраняются только в IndexedDB текущего браузера.

## Что уже работает

- карточки сотрудников со связанными наборами «эксплуатант — тип ВС — занимаемые кресла»;
- эксплуатанты КВП/АОН/АР и типы ВС A109, AW109, AW139, AS350, EC130, R44, R66, BO105 выбираются чекбоксами;
- редактирование и удаление сотрудников;
- учёт только свершившихся смен и других видов занятости без дополнительного статуса;
- восемь видов занятости: полётная смена, командировка, работа в офисе, периодическая и наземная подготовка, ожидание полёта, отпуск и выходной;
- для «Ожидания полёта» время начала и рабочее время не вводятся;
- для записи «Выходной» отдых считается от окончания предыдущего рабочего времени до начала следующего;
- несколько смен одной даты объединяются одной ячейкой даты в журнале;
- в месячном плане сотрудника с нужным допуском можно назначить на несколько разных бортов, а план выгрузить в оформленный Excel;
- командировка, отпуск и периодическая подготовка на произвольный период с отдельной записью каждого календарного дня;
- воскресенья внутри периода периодической подготовки автоматически отмечаются как выходные и остаются связанными с редактируемым периодом;
- для периодической подготовки время начала и рабочее время не вводятся; отдых на границе с полётной сменой принимается соответствующим установленным нормам и не формирует предупреждение;
- несколько полётов внутри одной смены;
- при отметке «Разделённая смена» форма раскрывает два полных набора данных — для первой и второй части; обе части связаны и считаются одной разделённой сменой в контроле отдыха;
- каждый полёт отображается отдельной строкой в едином журнале;
- выбор кресла «КВС» или «Пилот-инструктор» для каждого полёта;
- тип ВС выбирается из допусков сотрудника, после чего бортовой номер выбирается из зависимого списка для A109, AW109, BO105, R44, R66 и AS350;
- при создании новой смены сотрудник обязательно выбирается вручную и не подставляется автоматически;
- цели полёта: КВП, АОН, АР и АОН (УТП);
- ввод времени цифрами без обязательного двоеточия;
- рабочее, полётное и ночное время;
- расчёт окончания смены и отдыха от самой поздно завершившейся смены рабочего дня;
- редактирование и удаление смен;
- фильтры журнала по периоду и сотруднику, кнопки «Сегодня» и «Текущий месяц»;
- отдельная колонка примечаний в едином журнале;
- активные часы UTC и местного времени Сочи, Перми и Магадана;
- показатель количества выполненных полётных смен текущего месяца на главной;
- PDF-отчёт о налёте за произвольный период с таблицей «Кресло — Тип ВС — Цель — Налёт — Из них ночь»;
- PDF «Месячный отчёт»: отдельно по выбранному сотруднику либо по всему составу, включая дни без записей, типы ВС и полётное, инструкторское и ночное время;
- фирменный логотип Центра авиации «Солярис» в PDF-отчётах;
- контроль ежедневного отдыха не менее 12 часов;
- контроль еженедельного отдыха не менее 42 часов после шести рабочих дней;
- контроль отдыха не менее 48 часов после двух разделённых полётных смен подряд;
- пересечения смен не выводятся как нарушения в блоке контроля;
- раздел «Личные дела» для каждого пилота;
- налёт выбранного сотрудника за текущий месяц в шапке личного дела;
- отдельный раздел «Месячный план» после личных дел: матрица по всем бортам, датам месяца и ролям «Основной/Резерв»;
- единое окно «Запись занятости»: сотрудник, один день/произвольный набор дат/период, вид занятости и, для полёта, один или несколько доступных бортов;
- назначение в месячном плане только свободных сотрудников с допуском на соответствующий тип ВС;
- выходной, отпуск, подготовка и другая занятость блокируют назначение полёта с понятным объяснением причины; существующий полёт аналогично защищён от несовместимой записи;
- плановые назначения и занятость сразу отображаются в разделе «Полётные смены», откуда их можно удалить или одним нажатием открыть для изменения;
- отображение в месячном плане занятости, уже внесённой в журнал смен;
- импорт Excel-выгрузки «Сертификации» из Авиабит;
- предупреждения о сроках за 14 и 30 дней, просрочке и отсутствующих данных;
- ручное добавление и изменение записей личного дела;
- локальное автосохранение;
- резервная копия и восстановление в JSON.

## Публикация на GitHub Pages

1. Создайте новый репозиторий на GitHub, например `shtab-ls`.
2. Загрузите в него содержимое проекта и отправьте изменения в ветку `main`.
3. Откройте `Settings → Pages`.
4. В разделе `Build and deployment` выберите `GitHub Actions`.
5. Дождитесь выполнения процесса `Deploy to GitHub Pages` во вкладке `Actions`.

Файл `.github/workflows/pages.yml` уже содержит необходимую настройку публикации. При каждом обновлении ветки `main` сайт пересобирается автоматически.

## Работа с данными

После первого открытия:

1. Откройте `Личные дела` и нажмите `Загрузить Excel`.
2. Выберите выгрузку «Сертификации» из Авиабит. Сайт определит Ф. И. О., создаст карточку и покажет предварительный итог.
3. Нажмите `Импортировать`. При повторной загрузке обновятся только записи Авиабит, а ручные записи сохранятся.
4. Добавляйте полётные смены и проверяйте рассчитанное рабочее время, отдых и общий список предупреждений на главной.
5. Регулярно используйте команду `Скачать резервную копию`.

Excel обрабатывается прямо в браузере и никуда не отправляется. Для расчёта срока используется колонка `Конец`; пустая дата не рассчитывается автоматически и помечается как `Без срока` либо `Нет данных`.

Данные не входят в репозиторий и не отправляются на GitHub. Они привязаны к браузеру и устройству. Очистка данных сайта удалит локальную базу, поэтому резервное копирование обязательно.

## Локальный запуск для разработки

```bash
npm ci
npm run dev
```

Статическая сборка для GitHub Pages:

```bash
npm run build:github
```

Результат появится в каталоге `out`.

## Статус

Это первая рабочая версия. В контроль заложены согласованные базовые пороги приказа Минтранса России от 10 ноября 2025 г. № 381: 12, 42 и 48 часов. До переноса реальных производственных данных необходимо добавить шифрование локального хранилища и резервной копии, а также подтвердить применимость исключений и внутренних правил организации.
